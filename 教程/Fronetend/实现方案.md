# LangChain 前端教程最小实现方案

本文档基于同目录下的 `Frontend.md` 和 `Markdown_messages.md`，按你的当前选择重新制定落地计划：后端采用 Koa 框架，自己提供最小聊天 API；前端采用 React + Vite，通过 `fetch` 对接后端，并用 Markdown 组件渲染 AI 回复。

## 目标

完成后你会得到：

* 一个 Koa 后端 API：`POST http://localhost:3001/api/chat`。
* 一个 React 前端聊天页：`http://localhost:5173`。
* 前端可以发送用户消息，后端调用 LangChain Agent，返回 AI 文本。
* AI 文本按 Markdown 渲染，支持标题、列表、表格、代码块、引用和链接。

## 一、整理后端 Agent

先保证 `server/src/agents/agents1.ts` 导出 Agent，方便 Koa 路由调用：

```ts
import { createAgent } from "langchain";
import { MemorySaver } from "@langchain/langgraph";

export const agent = createAgent({
  model: "openai:gpt-5.4",
  tools: [],
  checkpointer: new MemorySaver(),
});
```

如果你后续要加天气、搜索、数据库查询等能力，只需要继续往 `tools` 数组里加工具。

## 二、配置后端环境变量

在 `server` 目录创建 `.env`：

```bash
OPENAI_API_KEY=你的 OpenAI API Key
```

如果使用其他模型供应商，就按对应 LangChain provider 的要求配置环境变量，并调整 Agent 中的 `model`。

## 三、安装 Koa 最小依赖

在 `server` 目录安装：

```bash
pnpm add koa @koa/router koa-bodyparser @koa/cors
pnpm add -D @types/koa @types/koa__router @types/koa-bodyparser @types/koa__cors
```

这些依赖分别负责：

* `koa`：HTTP 服务框架。
* `@koa/router`：声明 `/api/chat` 路由。
* `koa-bodyparser`：解析 JSON 请求体。
* `@koa/cors`：允许前端开发服务器跨域访问。

## 四、编写 Koa 聊天 API

用下面的最小实现替换或改造 `server/src/index.ts`：

```ts
import "dotenv/config";
import Koa from "koa";
import Router from "@koa/router";
import bodyParser from "koa-bodyparser";
import cors from "@koa/cors";
import { agent } from "./agents/agents1.ts";

type ChatMessage = {
  role: "system" | "user" | "assistant";
  content: string;
};

type ChatRequestBody = {
  messages?: ChatMessage[];
};

const app = new Koa();
const router = new Router();

app.use(
  cors({
    origin: "http://localhost:5173",
  })
);

app.use(bodyParser());

router.post("/api/chat", async (ctx) => {
  const body = ctx.request.body as ChatRequestBody;
  const messages = body.messages ?? [];

  if (messages.length === 0) {
    ctx.status = 400;
    ctx.body = { error: "messages 不能为空" };
    return;
  }

  const result = await agent.invoke({ messages });
  const lastMessage = result.messages.at(-1);
  const content = typeof lastMessage?.content === "string" ? lastMessage.content : "";

  ctx.type = "text/plain; charset=utf-8";
  ctx.body = content;
});

app.use(router.routes());
app.use(router.allowedMethods());

app.listen(3001, () => {
  console.log("Koa API running at http://localhost:3001");
});
```

这是最小可跑通版本：请求进来后一次性调用 `agent.invoke`，再把最后一条 AI 消息作为纯文本返回。前端先用这个版本完成对接，后续再升级为真正的 token 流式输出。

## 五、创建 React 前端

在仓库根目录创建前端项目：

```bash
pnpm create vite client --template react-ts
cd client
pnpm install
```

安装 Markdown 渲染依赖：

```bash
pnpm add react-markdown remark-gfm
```

React 使用 `react-markdown` 直接生成 React 元素，不需要 `dompurify`。如果以后改成 Vue、Svelte 或 Angular，并使用原始 HTML 渲染，就需要额外加入 `marked` 和 `dompurify`。

## 六、创建 Markdown 组件

新增 `client/src/components/Markdown.tsx`：

```tsx
import ReactMarkdown from "react-markdown";
import remarkGfm from "remark-gfm";

export function Markdown({ children }: { children: string }) {
  return (
    <div className="markdown-content">
      <ReactMarkdown remarkPlugins={[remarkGfm]}>{children}</ReactMarkdown>
    </div>
  );
}
```

`remark-gfm` 用来支持表格、删除线、任务列表和自动链接，这些都是 LLM 回复里很常见的 Markdown 格式。

## 七、创建聊天组件

新增 `client/src/components/Chat.tsx`：

```tsx
import { useState } from "react";
import { Markdown } from "./Markdown";

type ChatMessage = {
  id: string;
  role: "user" | "assistant";
  content: string;
};

const API_URL = "http://localhost:3001/api/chat";

export function Chat() {
  const [input, setInput] = useState("");
  const [messages, setMessages] = useState<ChatMessage[]>([]);
  const [isSending, setIsSending] = useState(false);

  async function handleSubmit(event: React.FormEvent<HTMLFormElement>) {
    event.preventDefault();

    const text = input.trim();
    if (!text || isSending) return;

    const userMessage: ChatMessage = {
      id: crypto.randomUUID(),
      role: "user",
      content: text,
    };

    const assistantMessage: ChatMessage = {
      id: crypto.randomUUID(),
      role: "assistant",
      content: "",
    };

    const nextMessages = [...messages, userMessage, assistantMessage];
    setMessages(nextMessages);
    setInput("");
    setIsSending(true);

    try {
      const response = await fetch(API_URL, {
        method: "POST",
        headers: {
          "Content-Type": "application/json",
        },
        body: JSON.stringify({
          messages: [...messages, userMessage].map((msg) => ({
            role: msg.role,
            content: msg.content,
          })),
        }),
      });

      if (!response.ok) {
        throw new Error(`请求失败：${response.status}`);
      }

      const content = await response.text();

      setMessages((current) =>
        current.map((msg) =>
          msg.id === assistantMessage.id ? { ...msg, content } : msg
        )
      );
    } catch (error) {
      setMessages((current) =>
        current.map((msg) =>
          msg.id === assistantMessage.id
            ? {
                ...msg,
                content:
                  error instanceof Error
                    ? error.message
                    : "请求失败：未知错误",
              }
            : msg
        )
      );
    } finally {
      setIsSending(false);
    }
  }

  return (
    <main className="chat-shell">
      <section className="message-list">
        {messages.map((msg) =>
          msg.role === "assistant" ? (
            <article className="message message-ai" key={msg.id}>
              <Markdown>{msg.content}</Markdown>
            </article>
          ) : (
            <article className="message message-human" key={msg.id}>
              {msg.content}
            </article>
          )
        )}
      </section>

      <form className="composer" onSubmit={handleSubmit}>
        <textarea
          value={input}
          onChange={(event) => setInput(event.target.value)}
          placeholder="输入消息..."
          rows={3}
        />
        <button type="submit" disabled={!input.trim() || isSending}>
          {isSending ? "发送中" : "发送"}
        </button>
      </form>
    </main>
  );
}
```

这一版先使用 `response.text()` 接收完整回复，代码更少、调试更简单。等 Koa API 跑通后，如果要升级为 token 流式输出，再把这一段替换成 `response.body.getReader()` 即可。

## 十一、最小落地顺序

1. 调整 `agents1.ts`，确保导出 `agent`。
2. 安装 Koa 相关依赖。
3. 用 Koa 实现 `POST /api/chat`。
4. 启动后端，用 Postman、curl 或浏览器前端验证 API 能返回文本。
5. 创建 React 前端和 `Markdown` 组件。
6. 接入 `fetch("http://localhost:3001/api/chat")`。
7. 最后补样式和 Markdown 富内容测试。

## 十二、后续升级方向

跑通最小版本后，再考虑这些增强：

* 把 `response.text()` 升级为 `ReadableStream`，实现 token 级流式显示。
* 在 Koa 中加入会话 ID，用来维护多轮对话。
* 增加统一错误处理中间件。
* 把 CORS origin 抽到 `.env`。
* 给 Agent 增加工具调用，并在前端展示工具执行状态。
